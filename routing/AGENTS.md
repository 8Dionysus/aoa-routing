# AGENTS.md

## Applies To

This card applies to `routing/` and every nested source-home path until a
nearer `AGENTS.md` narrows the lane.

## Role

`routing/` is the source-home district for `aoa-routing` owned navigation
behavior.

It owns thin-router source contours, routing-local policy seams, source-home
manifests, and implementation-adjacent contracts that are not participation in
a shared OS Abyss mechanic.

## Route Rules

- Put routing-local source behavior here before treating it as a mechanic.
- Route repeated AoA mechanic pressure to `mechanics/<head>/parts/<part>/`.
- Keep root generated surfaces under `generated/` only for public committed routing outputs that remain
  consumed outside one source-home or mechanic part.
- Keep root scripts under `scripts/` only for repo-wide validators, release gates, and public
  compatibility wrappers after implementation moves under source homes.
- Route former flat paths through the owning `PROVENANCE.md` and `legacy/`
  index instead of recreating root-path aliases.
- Keep this source home canonical until the G5 owner-switch receipt.
- After G5, reject new feature implementation here and route it to
  `aoa-sdk`; retain only compatibility, security, rollback, and deprecation
  changes until consumer-zero.

compatibility wrappers touched, mechanics handoffs preserved, checks run, and
