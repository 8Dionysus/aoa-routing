# Questbook Roadmap

## Current Contour

Questbook is present as routing-local support for source quest discovery,
quest-board reflection, and harvest fanout.

Source quest records live in `quests/`. `QUESTBOOK.md` is the human
open-obligation index. This mechanic owns operation law, source-only routing
seams, schemas, examples, validators, and generated hint posture.

## Next Work

- Keep `quests/routing/<state>/AOA-RT-Q-*.yaml` as the routing source-record
  layout.
- Keep `QUESTBOOK.md` aligned with open or reanchored records only.
- Keep source-only `inspect`, `expand`, and `handoff` as the active quest
  routing action set until upstream leaves justify more.
- Keep quest-board and RPG surfaces example-only until live builders exist.

## Review Routes

- Routing lane: use `quests/routing/` for repo-local source records.
- Agon lane: use `quests/agon/` for current Agon gate-routing follow-through
  records; keep historical Agon receipts in Agon legacy raw.
- Mechanic parts: use `parts/source-contract/`, `quest-routing-seam/`,
  `quest-board-seam/`, and `harvest-fanout/` for operation contracts.

## When Time Comes

- Add a quest catalog reader only if routing-local records need generated
  lookup separate from upstream source repos.
- Promote tiny-safe quest entry when live frontier d0/d1 r0/r1 leaves exist.
- Add lane README details when a lane becomes difficult to scan from
  `quests/README.md`.

## Out Of Scope

- Sibling quest meaning.
- Reward, progression, or acceptance authority.
- Parsing sibling live `quests/*.yaml` as source truth.
- Turning `QUESTBOOK.md` into a roadmap.
