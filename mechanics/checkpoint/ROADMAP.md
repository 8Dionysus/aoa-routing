# Checkpoint Roadmap

## Current Contour

Checkpoint is present as live-session reentry route review. The active part is
`live-session-reentry-route-review`.

The mechanic helps route interrupted or compacted work back to reviewable
state. It does not become memory truth, proof verdict, runtime scheduler, or
quest lifecycle authority.

## Next Work

- Keep reentry review route language aligned with source-home and mechanics
  topology.
- Add examples only when reentry pressure repeats enough to need source-shaped
  fixtures.

## When Time Comes

- Add owner-local checkpoint receipts after sibling owners accept local slices.
- Revisit release-support integration after checkpoint reentry creates
  repeated release-gate pressure.

## Out Of Scope

- Memory object storage.
- Proof verdicts.
- Hidden automation runners.
- Runtime scheduling.
