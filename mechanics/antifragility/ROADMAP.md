# Antifragility Roadmap

## Current Contour

Antifragility is present as stress routing, degraded route hints, composite
stress routing, rollback routing, quarantine routing, and via negativa.

The mechanic routes stress and repair pressure without becoming cleanup,
proof, memory, runtime, or playbook authority.

## Next Work

- Keep composite stress hints additive to existing router outputs.
- Keep rollback and quarantine route signals bounded to next-owner handoff.
- Keep via negativa docs as stop-lines, not broad doctrine spillover.

## When Time Comes

- Add stronger stress readers when repeated source-owned receipts justify a
  public generated route surface.
- Add owner-local repair receipts after evals, playbooks, memo, or runtime
  owners accept local slices.

## Out Of Scope

- One-score health.
- Deletion or cleanup authority.
- Runtime self-healing.
- Proof or memory truth.
