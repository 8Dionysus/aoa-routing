# Release-Support Roadmap

## Current Contour

Release Support is present as release gate routing, deployment ring routing,
installation routing, and watchtower escalation.

The mechanic helps route transition claims. It does not own release history,
runtime deployment truth, or sibling implementation release authority.

## Next Work

- Keep release gate routes aligned with root release checks and generated
  parity.
- Keep deployment, installation, and watchtower signals tied to owner evidence.
- Keep public claims out of generated outputs unless they are validated.

## When Time Comes

- Add release-support generated readers only if repeated transition signals
  need a public route model.
- Add stronger rollback/return integration after release-support and
  antifragility pressures repeat together.

## Out Of Scope

- Replacing root `CHANGELOG.md` or root `ROADMAP.md`.
- Runtime deployment authority.
- Public release claims without validation.
