# Recurrence Roadmap

## Current Contour

Recurrence is present as return navigation, downstream hints, and incident
reentry routing.

The mechanic points repeated or returning pressure back to source-owned
surfaces. It does not own retry policy, memory recurrence, or owner-local
observation truth.

## Next Work

- Keep return navigation and downstream hints additive to the thin router.
- Keep incident reentry weaker than proof, memory, and runtime owners.
- Update recurrence adapters only when owner-local recurrence manifests change.

## When Time Comes

- Add generated recurrence readers only when multiple recurrence parts need the
  same public route model.
- Add owner-local receipts after source repos accept recurrence handoffs.

## Out Of Scope

- Retry execution.
- Memory recall policy.
- Runtime recurrence scheduling.
