# Agon Gate Routing

This document defines the Wave V routing contour for Agon.

`aoa-routing` remains a thin navigation, typing, dispatch, and federation-entry orientation layer. It does not author meaning and it does not open arenas.

Wave V adds one additive surface:

`generated/agon_gate_routing_registry.min.json`

The surface helps a model recognize when ordinary service, thin routing, or routine cognitive handling is no longer enough and a pre-protocol Agon gate candidate should be raised.

## Routing may say

- ordinary service route is still enough
- owner review is required before Agon
- missing context blocks any honest gate decision
- a pre-protocol Agon gate candidate should be emitted
- a boundary breach should be quarantined or rerouted

## Routing may not say

- an arena session exists
- a sealed commit has happened
- a verdict has been issued
- a scar has been written
- retention has been scheduled
- an actor rank has changed
- closure is granted
- summon authority is granted
- a ToS promotion candidate exists

## Why this belongs in routing

The center owns Agon law.

Routing owns the small question:

> where should this go next?

Wave V adds a new possible answer:

> to an Agon gate candidate, under center-owned future law.
