## [0.3.0] - 2026-07-13

### Summary

- This release turns `aoa-routing` into a checked, source-home-backed thin
  routing provider: authored route inputs and mechanic-owned contracts are
  explicit, generated navigation stays derived, and negative/no-skill and
  overlay-budget behavior are enforced rather than implied.
- Public routing outputs now carry artifact identity and pass an OS Abyss
  evidence/subject-store trust path; the repo-local KAG index family and
  routing-local stats port remain bounded navigation/readout surfaces instead
  of becoming source, graph, or measurement authority.
- The release was rebuilt from Git rather than from the old `[Unreleased]`
  prose: all 43 first-parent commits from `v0.2.2` through `3d834e0` are
  accounted for below, spanning 826 changed paths and 39,630 additions / 3,629
  deletions. Only 8 of those 43 commits had touched this changelog before
  release preparation.

### Added

- Semantic and district-local `AGENTS.md` validation, with a slim root route
  card and preserved full reference surface.
- A portable checked `.agents/skills/` foundation plus owner-local quest,
  session-growth, diagnosis, and landing support surfaces used by routing
  workflows without transferring skill ownership here.
- A canonical decision-rationale lane with normalized `AOA-ROUTE-D-####`
  records, generated indexes, modeled-surface contracts, validators, and tests.
- A convex routing source home and package-local mechanics topology covering
  core routing, two-stage selection, boundary bridge, recurrence,
  antifragility, Agon, checkpoint, release support, Experience, Questbook,
  RPG, Titan, and adjacent routing-only heads.
- An active repo-local `evals/` port for routing evaluation pressure without
  central verdict authority.
- Artifact identity for federation entrypoints plus a thin-router trust profile,
  OS Abyss bundle/evidence promotion, materialized subject store, consumer
  trust gate, and fail-closed bundle coverage/admission checks.
- A local KAG provider handoff and canonical seven-index family for source
  surfaces, entities, artifacts, anchors, events, assertions, and relations.
- Added a root `stats/` port for the routing-owned question of how often
  current stats re-grounding hints send their primary inspect action directly
  to a non-stats owner, while keeping dispatch and owner truth outside the
  statistical packet.

### Changed

- Federation entrypoints and source pins were refreshed for the Pack5
  playbook contour, `/srv/AbyssOS` workspace shape, and center-map v2.
- Audit contracts, live dependency roots, memo mechanics refs, memory-route
  labels, and owner-local quest routes now point at current owner surfaces.
- Generated routing surfaces now follow local eval ports, assert explicit
  no-skill outcomes for negative cases, and apply token caps only after overlay
  scoring so the highest-value bounded routes survive.
- Decision records require valid dates; unmodeled decision lanes and non-list
  modeled surfaces fail generation/validation.
- Repo-local KAG publication is pinned, deterministic, lineage-aware,
  streamable, complete across all seven indexes, and stable across squash
  landing.
- Runnable validation and test commands stay in executable owners and nearest
  `AGENTS.md` route cards; the new release guide records publication shape
  without duplicating the command catalog.

- Re-grounded the copied aoa-stats catalog fixture and CI owner pin on the
  source-owned Surface Strength vocabulary route, and made fixture hydration
  resolve that catalog-declared authority ref instead of ignoring it.
- Re-grounded the authored composite stress-route examples on current
  mechanic-owned playbook, KAG, memo, and eval paths, then refreshed the
  stats-derived public hints after `aoa-stats` published the canonical proof
  and sibling refs. Validation now pins that owner revision; the advisory route
  verdict and routing authority ceiling are unchanged.
- Pinned validation to the aoa-stats owner ref that classifies the historical
  Runtime Closeout snapshot as contract history, removed its derived
  re-grounding hint, and preserved the generic Object, Repeated Window, and
  Source Coverage receipt routes.
- Pinned validation to the aoa-stats owner ref that retires the unsupported
  Titan Summon surface, then removed its derived re-grounding hint while
  preserving the active Titan Incarnation owner route.
- Pinned validation to the aoa-stats owner ref that retires the example-only
  Owner Landing aggregate, removed its derived re-grounding hint, and
  preserved the active Supersession Drop hint plus its aoa-skills and Dionysus
  owner-return routes.

### Fixed

- The stress-navigation schema now uses the canonical `aoa-routing` namespace.
- Composite stress, Stress Recovery, Titan Incarnation, Surface Strength, and
  generic receipt-backed owner routes were re-grounded on current owner refs;
  retired Titan Summon, Owner Landing, and Runtime Closeout hints were removed
  instead of kept alive as stale navigation.
- Routing artifact admission now rejects incomplete readmodel coverage,
  missing subject-store evidence, or absent consumer trust verdicts.

## [0.1.0] - 2026-04-01

First public baseline release of `aoa-routing` as the thin navigation and federation-entry layer in the AoA public surface.

This changelog entry uses the release-prep merge date.

### Summary

- first public baseline release of `aoa-routing` as the navigation and dispatch layer for AoA
- the public baseline now ships core routing outputs, pairing and return-navigation hints, federation-entry surfaces, and the optional two-stage low-context routing seam
- this release keeps source ownership explicit by routing to source-owned capsules, sections, and entry surfaces instead of recreating their meaning

### Added

- community-docs baseline established for this repository
- `CHANGELOG.md`, `CODE_OF_CONDUCT.md`, `SECURITY.md`, and `CONTRIBUTING.md`
- `mechanics/boundary-bridge/parts/federation-entry/docs/federation-entry-abi.md` as the doctrine surface for the federation entry orientation layer
- `generated/federation_entrypoints.min.json` and its schema-backed validation path
- additive `federation_queries` and `federation_starters` in `generated/tiny_model_entrypoints.json`
- build, validation, and CI support for `Agents-of-Abyss`, `Tree-of-Sophia`, `aoa-playbooks`, and `aoa-kag` as federation entry inputs
- narrow support for `Tree-of-Sophia/examples/tos_tiny_entry_route.example.json` as a source-owned ToS tiny-entry handoff surface

### Changed

- `tos-root` in `generated/federation_entrypoints.min.json` now hands off first to the source-owned ToS tiny-entry route while keeping `README.md` and `CHARTER.md` as root card surfaces
- `mechanics/boundary-bridge/parts/federation-entry/docs/federation-entry-abi.md`, `README.md`, and `ROADMAP.md` now describe the ToS tiny-entry sync as additive and non-authoritative
- `generated/federation_entrypoints.min.json` now publishes a second live `kag_view` for `Tree-of-Sophia`, while `kag-view-root` stays defaulted to `aoa-techniques`
- ToS tiny-entry input validation now accepts `bounded_hop` as the primary hop field and keeps `lineage_or_context_hop` as a compatibility alias during the current transition window
- memo recall hints now publish mode-indexed `capsule_surfaces_by_mode` for router-ready doctrine and parallel object-facing recall when upstream `aoa-memo` contracts expose a capsule step

### Included in this release

- core routing outputs under `generated/`, including `aoa_router.min.json`, `task_to_surface_hints.json`, `recommended_paths.min.json`, and `task_to_tier_hints.json`
- federation-entry and return-navigation surfaces under `generated/federation_entrypoints.min.json`, `generated/return_navigation_hints.min.json`, and `mechanics/boundary-bridge/parts/federation-entry/docs/federation-entry-abi.md`
- the optional low-context routing family under `generated/tiny_model_entrypoints.json` and the `two_stage_*` generated surfaces

### Validation

- Routing, rebuild-parity, two-stage, and test validation completed through the
  executable owner routes recorded in Git and CI history.

### Notes

- this release establishes routing as a navigation layer only; it does not make `aoa-routing` the authority surface for the repositories it reads
